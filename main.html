Rangkuman mengenai Git

Sistem Kontrol Versi (Version Control System - VCS): Git adalah salah satu jenis sistem kontrol versi yang digunakan untuk melacak perubahan dalam kode sumber proyek perangkat lunak.

Distribusi Terdistribusi: Git adalah sistem terdistribusi, yang berarti setiap pengguna memiliki salinan lengkap dari repositori proyek di komputer mereka sendiri. Ini memungkinkan kolaborasi dan pengembangan independen oleh banyak pengguna.

Commit: Dalam Git, perubahan dalam kode sumber disimpan dalam "commit." Setiap commit memiliki pesan yang menjelaskan perubahan yang dilakukan.

Branching: Git memungkinkan pengembang untuk membuat "branch" atau cabang baru dari kode sumber utama (biasanya disebut "master" atau "main"). Ini memungkinkan pengembangan fitur baru atau perbaikan bug tanpa mengganggu kode sumber utama.

Merge: Setelah pekerjaan di branch selesai, pengembang dapat menggabungkan (merge) cabang tersebut kembali ke branch utama. Ini menggabungkan perubahan kode sumber dari cabang ke branch utama.

Pull Request: Dalam kolaborasi tim, pengguna sering membuat "pull request" untuk mengajukan perubahan kode sumber dari cabang mereka ke branch utama proyek. Ini memungkinkan tim untuk mengulas perubahan sebelum mereka digabungkan.

Repositori: Repositori adalah tempat penyimpanan proyek Git. Ada repositori lokal (pada komputer pengguna) dan repositori jarak jauh yang biasanya di-host di platform seperti GitHub, GitLab, atau Bitbucket.

Konflik Merge: Terkadang, saat menggabungkan perubahan dari dua cabang yang berbeda, terjadi konflik merge. Ini terjadi ketika Git tidak dapat otomatis menggabungkan perubahan karena ada konflik dalam kode yang sama. Pengembang harus memecahkan konflik ini secara manual.

Stash: Git memungkinkan pengembang untuk "stash" perubahan sementara saat bekerja pada branch lain atau saat ada perubahan mendadak yang perlu disimpan.

Log dan Riwayat: Pengguna dapat melihat riwayat perubahan kode sumber menggunakan perintah "git log" untuk melacak siapa yang melakukan perubahan apa dan kapan.

.gitignore: File ".gitignore" digunakan untuk mengabaikan file atau direktori tertentu dalam repositori Git, seperti file sementara atau file yang tidak perlu.

Collaboration: Git memfasilitasi kolaborasi tim yang efisien, memungkinkan beberapa pengembang bekerja pada proyek yang sama tanpa mengganggu satu sama lain.

Git telah menjadi salah satu alat yang paling penting dalam pengembangan perangkat lunak modern, digunakan oleh pengembang di seluruh dunia untuk mengelola kode sumber mereka dengan efisien.
